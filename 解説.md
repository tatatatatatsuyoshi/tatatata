## 1. HTMLの基本構造

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>龍由 | Portfolio</title>
```

| コード | 意味 |
|---|---|
| `<!DOCTYPE html>` | 「これはHTML5で書かれたファイルです」とブラウザに宣言する |
| `lang="ja"` | このページは日本語だと宣言。検索エンジンや音声読み上げが正確に動く |
| `charset="UTF-8"` | 日本語などの文字を正しく表示するための文字コード指定 |
| `viewport` | スマホで表示したとき、画面幅に合わせてレイアウトを調整する |
| `<title>` | ブラウザのタブに表示されるタイトル |

## 2. 外部ライブラリの読み込み

```html
<script src="https://unpkg.com/react@18/umd/react.development.js"></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
```

Reactを使うために必要な3つのファイルをインターネットから読み込んでいます。

| ライブラリ | 役割 |
|---|---|
| `react.js` | Reactの本体。コンポーネント・状態管理などの機能が入っている |
| `react-dom.js` | ReactのコンポーネントをHTML（ブラウザの画面）に表示するための橋渡し役 |
| `babel.min.js` | JSX（`<div>` のようなHTML風の書き方）を普通のJavaScriptに変換する。これがないとブラウザはJSXを解釈できない |

> **ポイント:** 本来Reactはnpm（Node.js）でインストールするものですが、CDN（URL経由で読み込む方法）を使うことでインストール不要にしています。

## 3. CSS — 全体設定・カラー変数

### リセットCSS
```css
*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
```
すべての要素のデフォルトの余白（margin・padding）をゼロにして、ブラウザごとの見た目のブレをなくします。`box-sizing: border-box` はパディングを要素の幅に含める設定で、レイアウト計算が楽になります。

---

### カラー変数（:root）
```css
:root {
  --bg: #000014;      /* 最も暗い背景（ほぼ黒に近い深青） */
  --bg2: #00002a;     /* カードの背景色 */
  --bg3: #00003f;     /* アバターなど少し明るめの背景 */
  --cyan: #0077ff;    /* メインの明るい青 */
  --purple: #0033dd;  /* サブの深い青 */
  --pink: #00aaff;    /* ハイライト用の水色 */
  --text: #c8d8ff;    /* 本文テキストの色（青みがかった白） */
  --muted: #2255cc;   /* 補足テキストや薄い文字の色 */
}
```

`:root` とは「ページ全体のルート要素（htmlタグ）」のこと。ここに `--変数名: 値` の形で色を定義しておくと、CSSのどこでも `var(--cyan)` のように呼び出せます。色を一箇所で管理できるので、テーマ変更が簡単になります。

---

### body（基本スタイル）
```css
body {
  background: var(--bg);           /* 背景色 */
  color: var(--text);              /* 文字色 */
  font-family: 'Noto Sans JP', sans-serif; /* フォント */
  min-height: 100vh;               /* 最低でも画面の高さ100%を確保 */
  overflow-x: hidden;              /* 横スクロールを禁止 */
}
```

`100vh` の `vh` は「viewport height（画面の高さ）」の単位。`100vh = 画面の高さ100%` という意味です。

---

### スクロールバーのカスタマイズ
```css
::-webkit-scrollbar { width: 6px; }
::-webkit-scrollbar-track { background: var(--bg); }
::-webkit-scrollbar-thumb { background: var(--purple); border-radius: 3px; }
```

ブラウザの右端に出るスクロールバーを青色の細いデザインに変更しています。`-webkit-` は主にChromeやSafariで動くプレフィックスです。

## 4. CSS — 背景エフェクト（グリッド・ノイズ・グロー）

### ノイズオーバーレイ
```css
body::before {
  content: '';
  position: fixed;
  inset: 0;
  background-image: url("data:image/svg+xml,...");
  pointer-events: none;
  z-index: 9999;
}
```

`::before` は「bodyの前に擬似要素（見えない箱）を作る」という意味。画面全体を覆うノイズ（ザラザラしたテクスチャ）をSVGで生成して重ねています。`opacity: 0.03` なので微妙にしか見えないですが、これがあると画面に深みが出ます。`pointer-events: none` でマウス操作を素通りさせています。

---

### グリッド背景
```css
.grid-bg {
  position: fixed;
  inset: 0;
  background-image:
    linear-gradient(rgba(0,119,255,0.1) 1px, transparent 1px),
    linear-gradient(90deg, rgba(0,119,255,0.1) 1px, transparent 1px);
  background-size: 60px 60px;
  pointer-events: none;
}
```

縦横に1px幅の青い線を60px間隔で引いてグリッド（格子）を作っています。`linear-gradient` を縦と横（90deg）の2方向に重ねることで格子状になります。

---

### グローブロブ（光のにじみ）
```css
.blob { position: fixed; border-radius: 50%; filter: blur(100px); }
.blob1 { background: rgba(0,0,255,0.25); ... }
.blob2 { background: rgba(0,80,255,0.2); ... }
```

画面の左上と右下に大きな青い丸を置いて、`blur(100px)` でぼかして光のにじみ（グロー）を演出しています。実際には画面外にはみ出た状態なので、端だけがほんのり光って見えます。

---

## 5. CSS — ナビゲーションバー

```css
nav {
  position: fixed;      /* 画面上部に固定（スクロールしても動かない） */
  top: 0; left: 0; right: 0;
  z-index: 100;         /* 他の要素より前面に表示 */
  display: flex;
  justify-content: space-between; /* 左右に振り分け */
  align-items: center;
  backdrop-filter: blur(12px);    /* 背景をすりガラス風にぼかす */
  border-bottom: 1px solid rgba(0,119,255,0.3);
}
```

`backdrop-filter: blur()` がすりガラス効果の正体です。ナビゲーションの後ろにある要素がぼけて見える、モダンなUIでよく使われる手法です。

```css
.nav-logo {
  background: linear-gradient(135deg, var(--cyan), var(--purple));
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

テキストにグラデーションをかけるテクニック。背景にグラデーションを設定して、それを文字の形でクリッピング（切り抜き）することで文字自体がグラデーションに見えます。

## 6. CSS — ヒーローセクション

### レイアウト
```css
.hero {
  min-height: 100vh;   /* 画面1枚分の高さを確保 */
  display: flex;
  align-items: center; /* 垂直方向に中央揃え */
  justify-content: center;
}

.hero-inner {
  display: grid;
  grid-template-columns: 1fr 1fr; /* 左右2列に均等分割 */
  gap: 60px;
}
```

`1fr` の `fr` は「fraction（分数）」の単位。`1fr 1fr` で「残りのスペースを1:1で分割」という意味です。

---

### フェードインアニメーション
```css
.hero-text { animation: fadeInLeft 0.8s ease both; }

@keyframes fadeInLeft {
  from { opacity: 0; transform: translateX(-30px); }
  to   { opacity: 1; transform: translateX(0); }
}
```

`@keyframes` でアニメーションを定義します。`from`（開始）→`to`（終了）の変化を指定します。ここでは「左に30px ずれた透明な状態」から「元の位置で不透明」に変化させています。

---

### バッジとパルスアニメーション
```css
.hero-badge .dot {
  animation: pulse 2s ease infinite; /* 無限ループ */
}

@keyframes pulse {
  0%,100% { opacity:1; transform:scale(1); }
  50%      { opacity:0.4; transform:scale(0.8); }
}
```

「AVAILABLE FOR CHAT」の左側にある小さな丸が点滅しています。`0%→50%→100%` で大きさと透明度を変化させ、ゆっくり息をするように見えます。`infinite` で永遠にループします。

---

### レスポンシブなフォントサイズ
```css
font-size: clamp(2.8rem, 6vw, 4.5rem);
```

`clamp(最小値, 推奨値, 最大値)` は画面幅に応じてフォントサイズを自動調整する関数です。「最低2.8rem、画面幅の6%、最大4.5rem」という意味で、スマホでも大画面でも自然なサイズになります。

## 7. CSS — アバター（回転リング）

```css
.avatar-wrap {
  position: relative; /* 内部の要素をabsoluteで配置する基準になる */
  width: 240px;
  height: 240px;
}

.avatar-ring {
  position: absolute;
  inset: -16px;  /* 親より16px大きくはみ出す（リングの幅分） */
  border-radius: 50%;
  background: conic-gradient(from 0deg, #0077ff, #0000ff, #0033dd, #00aaff, #0000ff, #0077ff);
  animation: spin 4s linear infinite;
}

@keyframes spin { to { transform: rotate(360deg); } }

.avatar-ring-inner {
  position: absolute;
  inset: 3px;   /* リングの内側3pxを背景色で塗りつぶす（リング幅を作る） */
  border-radius: 50%;
  background: var(--bg);
}
```

**仕組みの解説:**
1. `avatar-ring`（外側の大きい円）にグラデーションを適用して回転させる
2. `avatar-ring-inner`（内側の小さい円）を背景色で塗りつぶして中を隠す
3. 結果として「グラデーションのリング部分だけ」が見えるリングになる

`conic-gradient` は中心から放射状ではなく、円周に沿って色が変わるグラデーションです。

```css
.avatar-circle::after {
  content: '';
  position: absolute;
  inset: 0;
  background: radial-gradient(circle at 35% 30%, rgba(0,119,255,0.35), transparent 60%);
}
```

アバター画像の上に半透明の青いグロー光を重ねています。`::after` 擬似要素を使うことで、HTMLを増やさずに光を追加できます。

## 8. CSS — カード・タグ・ステータス・スキル

### カードのホバーエフェクト
```css
.card {
  transition: transform 0.25s, border-color 0.25s, box-shadow 0.25s;
}

.card:hover {
  transform: translateY(-6px);              /* 上に6px浮かび上がる */
  border-color: rgba(0,119,255,0.7);        /* 枠が青く光る */
  box-shadow: 0 20px 60px rgba(0,51,255,0.35); /* 下に青い影 */
}
```

`transition` は「変化にかける時間」を指定します。0.25秒かけてなめらかに変化するので、カクつかずにホバーが演出できます。

---

### タグの色分け
```css
.tag-cyan  { background: rgba(0,119,255,0.15); color: var(--cyan); ... }
.tag-purple{ background: rgba(0,0,255,0.2);    color: #88aaff;     ... }
.tag-pink  { background: rgba(0,170,255,0.15); color: var(--pink); ... }
.tag-orange{ background: rgba(0,51,221,0.2);   color: #99ccff;     ... }
```

4種類のタグクラスを定義しています。今はテーマが全青なので、それぞれ微妙に色味の違う青になっています。

---

### ステータスボックスの光彩
```css
.status-box::before {
  content: '';
  position: absolute;
  top: -60px; right: -60px;
  width: 200px; height: 200px;
  border-radius: 50%;
  background: radial-gradient(circle, rgba(0,51,255,0.35), transparent 70%);
}
```

ステータスボックスの右上の角に青い光彩を追加しています。`overflow: hidden` と組み合わせることで、ボックスの外にはみ出た部分が隠れ、コーナーから光が漏れているように見えます。

## 9. CSS — レスポンシブ対応

```css
@media (max-width: 680px) {
  .nav-links { display: none; }               /* ナビリンクを非表示 */
  .hero-inner { grid-template-columns: 1fr; } /* 2列→1列に変更 */
  .hero-visual { order: -1; }                 /* アバターを上に移動 */
  .status-box { flex-direction: column; }     /* 横並び→縦並びに変更 */
}
```

`@media (max-width: 680px)` は「画面幅が680px以下のとき」に適用されるスタイルです。スマホは大体320〜430px程度なのでこのブロックが使われます。`order: -1` はFlexboxで表示順を変える機能で、HTMLの順番を変えずに見た目の順序だけ変えられます。

const skills = [

## 11. React — Typewriterコンポーネント

```jsx
function Typewriter({ text, speed = 80 }) {
  const [displayed, setDisplayed] = useState('');

  useEffect(() => {
    let i = 0;
    const id = setInterval(() => {
      setDisplayed(text.slice(0, ++i)); // 1文字ずつ増やす
      if (i >= text.length) clearInterval(id); // 全文字表示したら止める
    }, speed);
    return () => clearInterval(id); // クリーンアップ（メモリリーク防止）
  }, [text, speed]);

  return (
    <span>
      {displayed}
      <span style={{ borderRight: '2px solid #0077ff', ... }}>&nbsp;</span>
    </span>
  );
}
```

### 解説

- **`{ text, speed = 80 }`** — 引数の分割代入。`speed` はデフォルト値80を設定
- **`useState('')`** — 現在表示されている文字列を管理する「状態」。初期値は空文字
- **`useEffect`** — コンポーネントが画面に表示されたときに実行される処理
- **`setInterval`** — `speed` ミリ秒ごとに関数を繰り返し実行する
- **`text.slice(0, ++i)`** — 先頭から i 文字分を切り出す。`++i` で i を先に増やしてから使う
- **`clearInterval(id)`** — 全文字を表示したら繰り返しを止める
- **`return () => clearInterval(id)`** — コンポーネントが消えたときに後始末をする（これをしないとメモリが無駄に使われる）
- **カーソル部分** — `borderRight` で縦棒を表現し、タイプ中だけ表示（`opacity` で制御）

## 12. React — Appコンポーネント（画面の組み立て）

```jsx
function App() {
  const [mounted, setMounted] = useState(false);
  useEffect(() => { setTimeout(() => setMounted(true), 100); }, []);

  return (
    <>
      ...
    </>
  );
}
```

- **`<>...</>`** — Reactのフラグメント。複数の要素をまとめるとき、余分なdivを追加せずに済む書き方
- **`mounted` state** — コンポーネントが読み込まれたかどうかのフラグ（現在は定義されているが直接は使っていない、将来の拡張用）

---

### ナビゲーション
```jsx
<nav>
  <div className="nav-logo">TATSUYOSHI</div>
  <ul className="nav-links">
    <li><a href="#about">About</a></li>
    ...
  </ul>
</nav>
```

JSXでは `class` の代わりに `className` を使います（`class` はJavaScriptの予約語のため）。`href="#about"` はページ内の `id="about"` の要素にジャンプします。

---

### ヒーローセクション
```jsx
<h1>
  <span className="jp">龍由</span>
  <Typewriter text="Tatsuyoshi" speed={90} />
</h1>
```

`<Typewriter />` は自分で定義したコンポーネントを呼び出しています。`text` と `speed` を渡しています（これを「props（プロパティ）」と言います）。

---

### 趣味カードの自動生成
```jsx
{hobbies.map((h) => (
  <div className="card" key={h.title}>
    <div className="card-icon">{h.icon}</div>
    <div className="card-title">{h.title}</div>
    <div className="card-desc">{h.desc}</div>
    <span className={`card-tag ${h.tagClass}`}>{h.tag}</span>
  </div>
))}
```

`hobbies` 配列の `map()` メソッドで各要素をループしてカードを生成しています。`key={h.title}` はReactがリスト要素を管理するための識別子（一意である必要があります）。

バッククォート（`` ` ``）を使った `` `card-tag ${h.tagClass}` `` はテンプレートリテラルといって、文字列の中に変数を埋め込む書き方です。

---

### インラインスタイル（JSX内）
```jsx
<span style={{
  display: 'inline-block',
  background: 'rgba(0,0,255,0.25)',
  color: '#0077ff'
}}>2026</span>
```

JSXのインラインスタイルは `style={{ ... }}` と二重の `{}` で書きます。外側の `{}` はJSXの式、内側の `{}` はJavaScriptのオブジェクトです。プロパティ名はキャメルケース（`font-size` → `fontSize`）にします。

---

## 13. ReactDOM — 画面への描画

```js
ReactDOM.createRoot(document.getElementById('root')).render(<App />);
```

`<div id="root"></div>` というHTMLの空箱を取得して、そこにReactの `<App />` コンポーネント全体を描画しています。これが「ReactがHTMLを生成する」処理の起点です。

- **`document.getElementById('root')`** — id="root"の要素を取得
- **`createRoot()`** — React 18から導入された描画の初期化メソッド
- **`.render(<App />)`** — Appコンポーネントを描画

## まとめ — ファイルの全体構造

```
index.html
├── <head>
│   ├── メタ情報（文字コード、viewport、タイトル）
│   ├── 外部ライブラリ（React, ReactDOM, Babel）
│   └── <style> CSSスタイル
│       ├── カラー変数（:root）
│       ├── 背景エフェクト（グリッド・ノイズ・グロー）
│       ├── ナビゲーション
│       ├── ヒーロー・アバター
│       ├── カード・タグ・ステータス・スキル
│       └── レスポンシブ（スマホ対応）
│
└── <body>
    ├── 背景用のdiv（グリッド・ブロブ）
    ├── <div id="root"> ← ReactがここにHTMLを注入する
    └── <script type="text/babel"> Reactコード
        ├── データ定義（skills, hobbies）
        ├── Typewriterコンポーネント
        ├── Appコンポーネント（画面全体）
        └── ReactDOM.createRoot().render()
```
