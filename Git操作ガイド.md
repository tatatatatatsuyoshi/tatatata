# ローカル編集後にGitHubへアップロードする方法

---

## はじめに：Gitの基本的な考え方

Gitには「作業場所」が3つあります。

```
[ローカル作業ディレクトリ]  →  [ステージング]  →  [ローカルリポジトリ]  →  [GitHub（リモート）]
   ファイルを編集する場所       git add で追加      git commit で記録        git push で反映
```

- **ローカル作業ディレクトリ**：普段ファイルを編集している場所
- **ステージング（インデックス）**：「次のコミットに含める変更」を一時的に置く場所
- **ローカルリポジトリ**：コミット（変更の記録）が積み重なっていく場所
- **GitHub（リモートリポジトリ）**：インターネット上に公開・保存される場所

この3段階を経てGitHubに反映されます。

---

## 方法1：コマンドライン（ターミナル）

### 使い方の流れ

#### ステップ1：ターミナルを開く

- **Windows**：スタートメニューで「cmd」または「PowerShell」と検索して起動
- **VS Code内**：上メニューの「ターミナル」→「新しいターミナル」

#### ステップ2：プロジェクトのフォルダに移動する

```bash
cd "C:\Users\tmonn\OneDrive\デスクトップ\Claude\20260425"
```

※ `cd` は「Change Directory（ディレクトリを変更する）」の意味です。
※ すでにVS Codeのターミナルを使っている場合は自動でプロジェクトフォルダにいることが多いです。

#### ステップ3：変更内容を確認する（任意だが推奨）

```bash
git status
```

実行すると、変更されたファイルの一覧が表示されます。
- 赤色で表示されているファイル → まだステージングされていない変更
- 緑色で表示されているファイル → ステージング済みでコミット待ちの変更

#### ステップ4：ステージングする（git add）

変更したファイルを「コミットする準備」に加えます。

```bash
# すべての変更をステージングする場合（最もよく使う）
git add .

# 特定のファイルだけステージングする場合
git add src/components/Hero.jsx

# 特定のフォルダをステージングする場合
git add src/
```

`git add .` の `.` は「カレントディレクトリ以下すべて」という意味です。

#### ステップ5：コミットする（git commit）

変更内容に名前をつけて記録します。

```bash
git commit -m "変更内容のメモ"
```

`-m` の後の `""` の中にコミットメッセージを書きます。
日本語でも英語でも構いません。

**コミットメッセージの例：**
```bash
git commit -m "ヒーローセクションのデザインを修正"
git commit -m "プロフィール文を更新"
git commit -m "dragon.pngのパスを修正"
```

コミットメッセージは「何を変えたのか」が後から見てわかるように書くのがコツです。

#### ステップ6：GitHubへプッシュする（git push）

ローカルの記録をGitHubに送信します。

```bash
git push
```

初回だけ接続先を指定する必要がある場合は以下のように実行します：

```bash
git push -u origin main
```

2回目以降は `git push` だけでOKです。

---

### よく使うコマンド一覧

| コマンド | 意味 |
|----------|------|
| `git status` | 変更されたファイルの一覧を確認 |
| `git add .` | すべての変更をステージング |
| `git add ファイル名` | 特定ファイルをステージング |
| `git commit -m "メモ"` | 変更を記録 |
| `git push` | GitHubへ送信 |
| `git log --oneline` | コミット履歴を1行ずつ表示 |
| `git diff` | 変更内容の差分を表示 |

---

### コマンドを一度に実行する方法

毎回3つのコマンドを打つのが面倒な場合、`&&` でつなげて一度に実行できます：

```bash
git add . && git commit -m "変更内容のメモ" && git push
```

## 方法2：GitHub上で直接アップロード

GitHubのウェブサイト上で直接ファイルを編集・追加できます。
**小さな変更（1〜2ファイル）のときに便利ですが、ファイル数が多い場合は不向きです。**

### 既存ファイルを編集する場合

1. https://github.com/tatatatatatsuyoshi/tatatata を開く
2. 編集したいファイルをクリックして開く
3. 右上にある **鉛筆アイコン（Edit this file）** をクリック
4. 内容を編集する
5. 画面下の **"Commit changes"** ボタンをクリック
6. コミットメッセージを入力（省略可）して **"Commit changes"** を確定

### 新しいファイルをアップロードする場合

1. https://github.com/tatatatatatsuyoshi/tatatata を開く
2. **"Add file"** ボタンをクリック → **"Upload files"** を選択
3. ファイルをドラッグ＆ドロップ、またはクリックしてファイルを選択
4. 画面下の **"Commit changes"** をクリック

### 注意点

- フォルダごとアップロードはできない（ファイル単位）
- `node_modules` などの大量ファイルはアップロード不可
- 複数ファイルの変更には向いていない
- 基本的にはコマンドかVS Codeを使う方が確実

## 方法3：VS Code上でのプッシュ

VS Codeには**ソース管理（Git）機能**が内蔵されており、コマンドを使わずにGitの操作ができます。

### 使い方の流れ

#### ステップ1：ソース管理パネルを開く

左側のサイドバーにある **分岐アイコン**（Y字型のマーク）をクリックします。
またはショートカットキー `Ctrl + Shift + G` でも開けます。

#### ステップ2：変更ファイルを確認する

「変更」という項目の下に、編集したファイルが一覧表示されます。
ファイル名の右側には以下のアイコンが表示されます：

- `M`（Modified）→ 既存ファイルが変更された
- `U`（Untracked）→ 新しく追加されたファイル（まだGitに登録されていない）
- `D`（Deleted）→ 削除されたファイル

#### ステップ3：ステージングする（git add に相当）

- **すべてをステージングする場合**：「変更」という見出しの右にある **`+`マーク** をクリック
- **特定ファイルだけステージングする場合**：ファイル名にカーソルを合わせると現れる **`+`マーク** をクリック

ステージングされたファイルは「ステージングされた変更」という項目に移動します。

#### ステップ4：コミットメッセージを入力する

パネル上部のテキストボックス（「メッセージ」と薄く書かれている欄）に
コミットメッセージを入力します。

例：`ヒーローセクションのデザインを修正`

#### ステップ5：コミットする（git commit に相当）

テキストボックスの上にある **`✓ コミット`** ボタンをクリックします。

#### ステップ6：GitHubへプッシュする（git push に相当）

コミット後、パネル上部またはVS Code下部のステータスバーに
**「変更の同期」** または **「↑ 1 ↓ 0」** のようなボタンが表示されます。

それをクリックするとGitHubへプッシュされます。

または、パネル上部の **`...`（その他の操作）** をクリック →
**「プッシュ」** を選択してもOKです。

---

### VS Codeの便利な機能

- **差分の確認**：変更ファイルをクリックすると、変更前と変更後の差分がハイライト表示される
- **元に戻す**：ステージング前のファイルの右にある **矢印アイコン** をクリックすると変更を取り消せる
- **コミット履歴の確認**：拡張機能「Git Graph」や「GitLens」を入れると視覚的に履歴を確認できる

## どの方法を使えばいいか

| 状況 | おすすめ |
|------|----------|
| 普段の開発（複数ファイルを変更） | コマンドラインまたはVS Code |
| Gitコマンドに慣れていない | VS Code（視覚的でわかりやすい） |
| Gitに慣れてきた | コマンドライン（素早く操作できる） |
| 1ファイルだけ緊急で修正したい | GitHub上で直接編集 |
| ファイルをまとめて追加したい | GitHub上でアップロード（少数の場合） |

## 困ったときのよくあるエラーと対処法

### `git push` したら「rejected」と表示された

GitHub上に自分のローカルにない変更がある場合に起こります。
まず `git pull` でGitHubの変更を取り込んでから、再度 `git push` してください。

```bash
git pull
git push
```

### `git push` でパスワードを求められた

GitHubは現在パスワード認証に対応していません。
**Personal Access Token（PAT）** を作成して使う必要があります。

1. GitHub → 右上のアイコン → Settings
2. 左メニュー最下部の **"Developer settings"**
3. **"Personal access tokens"** → **"Tokens (classic)"**
4. **"Generate new token"** → 有効期限とスコープ（`repo` にチェック）を設定
5. 生成されたトークンをパスワードの代わりに使用

### `git add .` してから間違いに気づいた

ステージングを取り消す（コミットはしていない場合）：

```bash
git reset HEAD .
```

### コミットしたけど内容を間違えた（まだpushしていない場合）

直前のコミットメッセージを修正する：

```bash
git commit --amend -m "正しいメッセージ"
```

## このプロジェクトの場合のまとめ

このプロジェクト（tatatatatatsuyoshi/tatatata）では、以下の手順でOKです：

```bash
# 1. 変更確認
git status

# 2. ステージング
git add .

# 3. コミット
git commit -m "変更内容のメモ"

# 4. プッシュ
git push
```

GitHub Actionsが自動的にビルド＆デプロイを行うため、
`git push` 後、1〜2分待てばGitHub Pagesに反映されます。

デプロイの状況は https://github.com/tatatatatatsuyoshi/tatatata/actions で確認できます。
